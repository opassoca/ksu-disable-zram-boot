# ⚡ Disable ZRAM on Boot (Модуль KernelSU / Magisk)

---

Низколатентный модуль утилит ядра, разработанный для **KernelSU и Magisk**, предназначенный для отключения своп-раздела ZRAM (`/dev/block/zram0`) во время запуска Android. Идеально подходит для высокопроизводительных устройств, чтобы устранить нагрузку на сжатие со стороны процессора, уменьшить системные микро-фризы и оптимизировать расход батареи в тяжелом режиме многозадачности.

---

## ⚡ Как это работает

По умолчанию система Android подключает и активирует своп ZRAM сразу после загрузки. Этот модуль безопасно перехватывает процесс в два этапа:

1.  **post-fs-data.sh (Ранняя фаза):** выполняет команду `swapoff` на `zram0` до того, как сжатые страницы памяти начнут заполняться.
2.  **service.sh (Фаза подтверждения):** ожидает, пока системное свойство `sys.boot_completed` не станет равным `1` (что указывает на полную загрузку пользовательского интерфейса) для проверки стабильности загрузки.

---

## 🛡️ Защитный сторожевой таймер (Anti-Bootloop Watchdog)

Раннее отключение системного свопа на некоторых сильно модифицированных прошивках может привести к нехватке памяти (Out-Of-Memory) и циклической перезагрузке (bootloop). Чтобы предотвратить это, в модуль встроен **автономный сторожевой таймер**:
*   Увеличивает счетчик загрузок, сохраненный в `/data/adb/zram_disable_count` при запуске.
*   Сбрасывает счетчик на `0` сразу после успешного завершения загрузки (`sys.boot_completed=1`).
*   Если система не завершает загрузку полностью **3 раза подряд**, модуль **автоматически отключает себя** при последующих запусках для восстановления настроек ZRAM по умолчанию и безопасного восстановления системы.

---

## 📂 Структура модуля

*   `module.prop`: Метаданные модуля.
*   `post-fs-data.sh`: Скрипт раннего запуска с правами суперпользователя для отключения свопа.
*   `service.sh`: Скрипт проверки стабильной загрузки и сброса счетчика таймера.
*   `index.html`: Интерактивный симулятор загрузки с консолью логов Android и панелью статуса ZRAM.

---

## 🛠️ Установка

1.  Заархивируйте все файлы в папке в один zip-файл:
    ```bash
    zip -r disable-zram-boot.zip . -x "*.git*" "index.html" "server.py"
    ```
2.  Установите полученный файл `.zip` через приложение **KernelSU**, **APatch** или **Magisk Manager**.
3.  Перезагрузите устройство.

---


## 📱 Интеграция и мониторинг в Termux
Полностью совместим с средой Android Termux. Вы можете отслеживать статус ZRAM и логи загрузки, выполнив:
```bash
su -c "cat /data/adb/disable_zram.log"
```
